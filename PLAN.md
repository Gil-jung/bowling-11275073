# PLAN — Cycle 6: 퍼펙트 게임 (300점)

## 0. 이전 사이클

- Cycle 1(거터 게임) — 완료
- Cycle 2(스페어 보너스) — 완료
- Cycle 3(스트라이크 보너스) — 완료
- Cycle 4(10프레임 스트라이크 보너스 확인) — 완료, 코드 변경 없이 GREEN
- Cycle 5(10프레임 스페어 보너스 확인) — 완료, 코드 변경 없이 GREEN

## 1. 목표 (RED 1단계)

**단 하나의 동작**: 12번 모두 스트라이크(퍼펙트 게임)를 치면 총점은 `300`이다.
이는 CLAUDE.md에 명시된 kata의 대표적인 엣지 케이스이며, 남은 마지막 시나리오다.

## 2. 작성할 실패 테스트 — 그리고 사전 고지

- 파일: `test_bowling.py`
- 테스트: `test_perfect_game_scores_300`

```python
def test_perfect_game_scores_300():
    game = Game()
    for _ in range(12):
        game.roll(10)

    assert game.score() == 300
```

**사전 고지**: Cycle 4·5와 같은 이유로, 프레임 1~9의 스트라이크는 각각
`10 + 다음 2롤(=10+10)`로 계산되고, 10프레임의 스트라이크 + 보너스 2롤도
이미 올바르게 처리됨을 확인했다. 따라서 이 테스트도 **코드 변경 없이 이미
통과할 가능성이 높다.** 실제 실행 결과를 직접 확인해 RED/GREEN 여부와 그
이유를 판단한다.

- 만약 이미 GREEN이면: 프로덕션 코드 변경 없이, 회귀 방지용 확인 테스트로만
  커밋한다. 이로써 CLAUDE.md에 명시된 kata의 모든 시나리오(오픈 프레임, 스페어,
  스트라이크, 퍼펙트 게임, 10프레임 엣지 케이스)가 테스트로 커버된다.
- 만약 실패한다면: 그 실패 원인에 따라 GREEN 구현을 진행한다.

## 3. 구현 방향 (필요할 경우에만, GREEN에서 진행)

테스트가 실제로 실패할 경우에만 대응한다. 예상 가능한 원인은 없으나, 만약
실패한다면 원인을 분석해 최소한으로 수정한다.
